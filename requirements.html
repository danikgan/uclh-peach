Requirement and Scope


Priorities list

  • Form Renderer
  • Design Guide
  • Visual Report
  • Dashboard

    • The following MoSCoW is including all the requirements for each of the project mixed. This is because the development of the projects is running simultaneous, therefore, priorities have to be taken into account, whether certain feature in one project outweighs the importance of some other features in a different project.


    MoSCoW:
    Must Have

  • Design and create a front-end user interface that retrieves information from a server and displays it in a form
  • Design the web-form
  • Add buttons to submit / receive new information
  • Negotiate the use of desired libraries and tools for greater maintenance by other teams (future or existent)
  • Create, update and maintain a Design Guide for the Peach project
  • Create the website
  • Add components from the original React-Bootstrap website
  • Make use of the existing functionality done by previous teams
  • Establish the general framework for everyone to use when updating the website
  • Maintain the website when other teams are updating it


    • Should Have

  • Change contradicting styles on the Design Guide accommodating the general look of the website (the code copied from the official React-Bootstrap website has a different style)
  • Debug the website (and contact the client about further tasks)
  • An interface which would allow to easily edit information before displaying it in the Form Renderer -- to be further discussed with Team 41 and the client


    • Could Have

  • Visual Reporting application which simplifies the reporting of medical information -- to be further discussed with the client for greater clarifications of the details
  • Visual representation of large amount of data, using D3.js -- to be further confirmed with the client


    • Won't Have

  • Any login system for all the websites produced

    UI requirements

    This section includes UI requirements for the Design Guide and Form Renderer projects separated. This is due to the fact that functional / non-functional requirements for each project must be separated to evaluate every project uniquely.


    Design Guide
    Functional

  • Must have React (or similar) framework compatibility
  • Must be responsive and optimised for each screen resolution or device

    • Non-functional

  • Quick loading of the information


    • Form Renderer
    Functional

  • Must conform to professional medical color guidelines
  • Must show when loading is complete
  • Must display when the information has been submitted successfully or not

    • Non-functional

  • Retrieving of the information as well as receiving to be as quick as possible
